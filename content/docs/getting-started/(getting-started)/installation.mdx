## Installation Overview

You can install Aegis using either **standard Python 3.12.10** or **UV**.  

- **New or beginner users:** Use standard Python for simplicity.  
- **Advanced users or those who prefer faster package management:** UV is optional and can simplify dependency installation.

Below, we’ll walk through the installation process while noting differences where applicable.


## Prerequisites

Check your Python version:

<Tabs items={['Mac/Linux', 'Windows']}>

<Tab value="Mac/Linux">

```bash
python3 --version
```

or

```bash
python3.12 --version
```

</Tab>

<Tab value="Windows">

```bash
python --version
```

or

```bash
py -3.12 --version
```

</Tab>

</Tabs>

If the output doesn't show Python `3.12.10`, follow the [Python Setup Guide](../../../guides/python)
or install `uv` from the [uv installation page](https://docs.astral.sh/uv/getting-started/installation/).

<Steps>

### Create an Aegis Directory

You can create this folder using either your file explorer or the terminal. Name it whatever you like.
Just make sure the folder name does not contain spaces, as this can sometimes cause issues with the client. 

Examples:

```bash
mkdir aegis
```

or 

```bash
mkdir cpsc-383-aegis
```

Run all the commands shown below inside aegis directory you created.

### Create a Virtual Environment

Virtual environments isolate project dependencies from your system Python.

**Python Users**

<Tabs items={['Mac/Linux', 'Windows']}>

<Tab value="Mac/Linux">

Use `python3` if it points to version 3.12, otherwise specify `python3.12`:

```bash
python3 -m venv .venv
```

or

```bash
python3.12 -m venv .venv
```

</Tab>

<Tab value="Windows">

Use `python` if it points to version 3.12, otherwise specify `py -3.12`:

```bash
python -m venv .venv
```

or

```bash
py -m venv .venv
```

</Tab>

</Tabs>

**UV Users**

```bash
uv venv --python 3.12 .venv
```

### Activate the Virtual Environment

```bash tab="Mac/Linux"
source .venv/bin/activate
```

```bash tab="Windows"
.\.venv\Scripts\Activate.ps1
```

Once activated, you can use the `python` executable (it will point to the venv’s interpreter).

### Download Aegis 

**Python Users**

```bash
pip install aegis-game
```

**UV Users**

```bash
uv pip install aegis-game
```

### Initialize

```bash
aegis init
```

If you are initializing the mas assignment use:

```bash
aegis init --type mas
```

<Callout type="warning">

If the client fails to download, you can manually download the latest release
from [Aegis GitHub Releases](https://github.com/AEGIS-GAME/aegis/releases).

Look for the latest `aegis-client: vx.x.x` tag and download the client for your operating system.

Make sure you also add the client app/executable under the `client` folder show below.

</Callout>

### Update Stub File

The `aegis init` command creates a default `config.yaml`.
If your professor provided a modified `config.yaml`, replace the default with their 
version before proceeding.

Next, run:

```bash
aegis forge
```

This updates your stub file for the assignment, providing proper LSP typing and hiding unnecessary functions,
so you can focus only on what’s required.
